# Personality
[tono, nivel de detalle y estilo de colaboración]

# Context
[contexto real disponible: código, error, tablas, versión, entorno, reglas de negocio]

# Goal
[resultado concreto y visible que debe entregar el modelo]

# Success criteria
- [criterio verificable 1]
- [criterio verificable 2]
- [criterio verificable 3]
- [criterio verificable 4]

# Constraints
- [restricción técnica]
- [restricción de seguridad]
- [restricción de compatibilidad]
- [restricción de negocio]
- [restricción de dependencias o versiones]

# Evidence rules
- Basarse solo en el contexto proporcionado
- No inventar tablas, endpoints, librerías ni credenciales
- Si falta evidencia, declarar supuestos o pedir el dato mínimo necesario

# Output
- [sección requerida 1]
- [sección requerida 2]
- [sección requerida 3]
- [longitud o nivel de detalle esperado]

# Validation
- [cómo probar o verificar la respuesta]
- [comandos, checks, tests o revisión manual esperada]

# Stop rules
- Si falta información crítica, pedir aclaración
- Si hay ambigüedad menor, declarar supuestos y continuar
- Si la tarea puede causar efectos secundarios peligrosos, advertir antes de proponer ejecución
- Si no hay suficiente información para una solución confiable, explicar qué falta
